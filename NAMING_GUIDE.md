# NAMING_GUIDE

> 파이프라인 트래커 일감 네이밍 + 의존성 연결 1페이지 가이드. 파트장 대상. (v1.0 / 2026-04-22)

## 1. 한 줄 규칙

```
일감 제목 = [대상] [동사]
```

- **대상**은 선택 (`폴리싱`처럼 동사만 가능)
- **동사**는 아래 8개 중 하나로 끝나야 함
- 괄호 `( )`, em dash `—`, 이모지, 전각 괄호 금지
- 3~60자

## 2. 허용 동사 (action_verb) 8개

| 동사 (label) | code | 언제 쓰나 | 예시 |
|-------------|------|----------|------|
| 구현 기획서 작성 | `draft_doc` | G3 단계, 구현용 기획 문서 초안 | `하렌홀의 비명 구현 기획서 작성` |
| 구현 | `implement` | G5 단계, 실제 게임 내 구현 | `메인퀘스트 보스 - 쉘윈 구현` |
| 데이터 작업 | `data_entry` | 테이블 데이터 입력/조정 | `보스 스킬 데이터 작업` |
| 리소스 요청 | `resource_request` | G4, 아트/사운드 발주 | `하렌홀의 비명 리소스 요청` |
| 폴리싱 | `polish` | G7 직전, 품질 개선 | `챕터4.1 보스 패턴 폴리싱` |
| QA 대응 | `qa_response` | G7, QA 이슈 대응 | `메인퀘스트 보스 QA 대응` |
| 재미검증 | `fun_review` | G6 단계 | `시즌5 던전 재미검증` |
| IP 검수 대응 | `ip_review` | G0a, IP 홀더 피드백 | `챕터5 시나리오 IP 검수 대응` |

새 동사가 필요하면 `task_action_verbs` 테이블에 INSERT (마이그레이션 불필요).

## 3. Do / Don't

| Don't | Do |
|-------|-----|
| 챕터4.1 메인 퀘스트 보스(쉘윈) | 챕터4.1 메인 퀘스트 보스 - 쉘윈 구현 |
| 하렌홀 (PvPvE) 작업 | 하렌홀의 비명 구현 |
| 폴리싱 필요함 | 폴리싱 |
| 보스 — 자일스 | 보스 - 자일스 구현 |
| 구현기획서작성 | 구현 기획서 작성 (띄어쓰기 필수) |
| 기획서 쓰기 | 구현 기획서 작성 (정규 동사만 허용) |
| 폴리싱 🎨 | 폴리싱 (이모지 금지) |

## 4. 같은 콘텐츠의 파생 업무는 템플릿으로

콘텐츠 생성 시 파트장이 템플릿을 선택하면 일감 4~5개가 자동 생성되고 선행/후행 관계도 자동으로 연결됩니다.

기본 템플릿 (확장 가능):

| code | 파트 | 스텝 |
|------|-----|-----|
| `main_quest_boss` | 전투 | 기획서(5일) → 구현(10일) → 폴리싱(3일) → QA(4일) |
| `field_dungeon` | 필드 | 기획서(5일) → 리소스(2일) → 구현(12일) → 폴리싱(4일) → QA(5일) |
| `quest_main` | 퀘스트 | 기획서(4일) → 구현(8일) → 폴리싱(2일) → QA(3일) |

수동으로 의존성 걸 때는 아래 2종만 씀:

- **선행 (finishes_before)**: A 끝나야 B 시작. JIRA `is blocked by`. 간트에 실선 화살표
- **동시 (starts_together)**: A와 B 함께 시작. JIRA `relates to`. 간트에 점선 화살표

## 5. Row 값 표준

### 목표
`론칭` / `1차 업데이트` / `시즌1` ~ `시즌10` / `CBT` 중 하나. 공백은 `미정`으로 자동 채움. `1차`는 `1차 업데이트`로 자동 교정.

### 지역/챕터
`3.1. 스톰스엔드` 형식 (code + 이름). `3.1` code만 입력해도 시스템이 `display`로 결합.

### 이슈 타입
`Task` / `Sub-Task`. 부모 일감 있으면 자동 `Sub-Task`.

### 유관파트
`field` / `quest` / `combat` / `level` / `art` / `narrative` / `ux` / `class` / `validation`

복수는 `,` + 공백: `field, art`. 슬래시(`/`)도 수용.

### 담당자 복수
쉼표 + 공백: `곽명준, 안태경` (슬래시 금지)

## 6. 위반 시

- xlsx에 잘못 적어도 웹앱 import는 작동. 다만 정합성 체크 리포트에 경고가 뜸
- action_verb NULL인 task는 리포트 목록에 뜸 → 이 가이드 참고해서 동사 배정
- 괄호 있는 제목은 JIRA 자동 생성에서 부분적으로 파싱 실패 → 하이픈으로 교체
